# Add DATABASE_URL to crossify-platform Service

## Problem
The `DATABASE_URL` environment variable is not set in the crossify-platform service, so it's using SQLite instead of PostgreSQL.

## Solution

### Step 1: Get DATABASE_URL from Postgres Service

1. In Railway, click on the **Postgres** service (left sidebar)
2. Click on the **Variables** tab
3. Find the `DATABASE_URL` variable
4. Copy its value (it will look like: `postgresql://postgres:password@hostname:5432/railway`)

### Step 2: Add DATABASE_URL to crossify-platform Service

1. Click on the **crossify-platform** service (left sidebar)
2. Click on the **Variables** tab
3. Click **"+ New Variable"** button (usually at the top right)
4. Enter:
   - **Name:** `DATABASE_URL`
   - **Value:** (paste the value you copied from Postgres service)
5. Click **"Add"** or **"Save"**

### Step 3: Redeploy Backend

After adding the variable, Railway should automatically redeploy. If not:
1. Go to **crossify-platform** service ‚Üí **Deployments** tab
2. Click the three dots (‚ãØ) on the latest deployment
3. Click **"Redeploy"**

## Alternative: Railway Variable Reference

If Railway supports variable references (some platforms do):
1. In **crossify-platform** ‚Üí **Variables**
2. Click **"+ New Variable"**
3. Instead of pasting the value, look for a **"Reference"** or **"Add Reference"** option
4. Select the **Postgres** service and the `DATABASE_URL` variable
5. This will automatically sync the value

## Troubleshooting

### If DATABASE_URL is not visible in Postgres service:
- Check that Postgres service is running
- Check the **Settings** tab of Postgres service
- The DATABASE_URL should be automatically generated by Railway

### If backend still uses SQLite after adding DATABASE_URL:
- Verify the variable name is exactly `DATABASE_URL` (case-sensitive)
- Check that the value starts with `postgresql://`
- Verify the backend service has been redeployed
- Check logs for database initialization messages
